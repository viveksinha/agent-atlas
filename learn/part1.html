A. What is an LLM?

LLM = Large Language Model. It is an AI trained on massive amounts of text (books, websites, code) to understand and generate human language.

Think of it as an extremely well-read assistant who can write, explain, summarize, and answer questions — but has no memory of past conversations unless you give it context.

Examples: ChatGPT (OpenAI), Claude (Anthropic), Gemini (Google).

Simple analogy: Autocomplete on your phone, but 1000x smarter. It predicts the most useful next word, sentence, or paragraph based on what you typed.

How an LLM works (pipeline)

flowchart LR
    A["Training data\n(books, websites, code)"] --> B[LLM]
    C[Your prompt] --> B
    B --> D[Generated response]

    style B fill:#e8f4fc,stroke:#333

Each arrow is one step: the model was trained on text, you send a prompt, it predicts the best continuation word by word until the answer is complete.

B. What is GitHub Copilot?

GitHub Copilot is an AI coding assistant built into your code editor. It reads what you're typing and suggests the next line, function, or even entire blocks of code.

It is powered by an LLM trained specifically on public code from GitHub. Other such tools are Cursor or Claude Code.

Where does it live?

  • VS Code — the most popular free code editor by Microsoft. Copilot shows up as a side panel or inline suggestion as you type.
  • GitHub.com — the website where developers store and share code. Copilot can answer questions about a repo directly on the site.

Example: You type # function to sort a list of names alphabetically and Copilot writes the full code for you.

Where Copilot sits in the stack

flowchart LR
    You[You type in editor] --> Editor[VS Code / Cursor]
    Editor -->|sends context| LLM[Code-trained LLM]
    LLM -->|returns suggestion| Editor
    Editor --> You

    style LLM fill:#e8f4fc,stroke:#333

You stay in the editor. The LLM never runs on your machine — it reads your code snippet and sends back a suggestion you accept or ignore.

Why you'll see .md (Markdown) files

In VS Code, Cursor, and most agent setups, instructions and documents are saved as Markdown files — plain text with simple formatting (headings, bullets, bold). The extension is .md.

Why Markdown?What it means for you
LLMs read it easilyPlain text — no converting Word docs or PDFs
StructuredHeadings and numbered steps help the agent follow a skill
Built into VS Code / CursorOpen the file, use Preview (or Cmd+Shift+V / Ctrl+Shift+V) to see formatted output
Easy to share and versionStore skills and notes in Git; see exactly what changed

You'll use Markdown for skills, project notes, and README-style docs. Part 2, Section N goes deeper when you build your first skill.

C. What is Agentic AI? What is an Agent?

An agent is an AI that doesn't just answer questions — it takes actions to complete a goal.

ChatbotAgent
You ask"How do I send an email?""Send an email to my team with the meeting summary"
It doesExplains the stepsActually sends the email

An agent can:

  • Use tools (search the web, read files, call APIs)
  • Make a plan (break a big task into steps)
  • Execute the steps one by one
  • Loop back and fix mistakes

Agentic AI is the broader idea of AI systems that act autonomously — not just respond.

Example in Cursor: You say "Find all bugs in my code and fix them." The agent reads your files, identifies issues, edits the code, and tells you what it changed.

Chatbot vs agent (flow)

flowchart TB
    subgraph chatbot [Chatbot]
        Q1[You ask a question] --> A1[AI explains the steps]
    end

    subgraph agent [Agent]
        Q2[You give a goal] --> P[Plan steps]
        P --> T[Use tools]
        T --> O[Observe results]
        O --> D{Done?}
        D -->|No| P
        D -->|Yes| R[Delivers outcome]
    end

The chatbot stops at advice. The agent loops — plan, act, check — until the job is actually done.

D. How to Write a Good Prompt

A prompt is the instruction you give to an AI. Better prompts → better results.

The 4-part formula

PartWhat it doesExample
RoleTell the AI who to be"You are a competitive intelligence analyst at a consulting firm…"
TaskState exactly what you want"…write a one-page competitor landscape brief…"
ContextGive relevant background"…covering three direct rivals in the enterprise payments API market…"
FormatSpecify the output structure"…in bullet points with a summary at the top."

The 4-part formula (pipeline)

flowchart LR
    R["① Role\nwho to be"] --> T["② Task\nwhat to do"]
    T --> C["③ Context\nbackground"]
    C --> F["④ Format\noutput shape"]
    F --> Out[Better AI output]

    style Out fill:#d4edda,stroke:#333

Think of it as a pipe: each part feeds the next. Skip Role and the tone may be wrong. Skip Context and the answer will be generic. Skip Format and you'll get a wall of text.

Good vs Bad

Bad promptGood prompt
"Write something about payments""Write a 3-sentence explanation of how Stripe handles failed payments, for a non-technical audience."
"Summarize the competitor""As a business analyst, summarize Acme Corp's Q3 earnings call in 5 bullets — focus on pricing and partnerships."
"Fix my data""This SQL query returns duplicate rows on the region join. Find the bug and explain why it happens."

Rule of thumb: If you wouldn't give this instruction to a new intern and expect the right result, rewrite it.

E. Skills, Rules, and Instructions

These are the three ways you customize how an AI agent behaves.

Instructions

Instructions guide the behavior of the agent. Set once, always active.

Example: "Always respond in plain English. Never use jargon. When unsure, ask a clarifying question."

Rules

Guardrails — things the agent must always do or never do.

Example: "Never post to Microsoft Teams without the user explicitly asking." or "Always check for duplicate Jira tickets before creating a new one."

Skills

Reusable playbooks for specific tasks. A skill is a Markdown file (.md) that tells the agent **how to do something** step by step — like a standard operating procedure (SOP).

Example: A "competitive pulse brief" skill tells the agent: pull the competitor name → find the key market move → write 3 summary angles → pick the clearest one for executives.

Analogy: Instructions = employee handbook. Rules = company policies. Skills = job-specific training manuals.

How the three layers stack

flowchart TB
    subgraph always_on [Always loaded]
        I[Instructions\nbaseline personality]
        R[Rules\nhard guardrails]
    end

    subgraph on_demand [Loaded when task matches]
        S[Skills\nstep-by-step playbooks]
    end

    I --> Agent[Agent behavior]
    R --> Agent
    S --> Agent

    style Agent fill:#e8f4fc,stroke:#333

Instructions and rules apply to every task. Skills kick in only when the job matches — like pulling the right manual off the shelf.

F. Modes: Plan, Ask, Agent

In tools like Cursor, the AI operates in different modes depending on what you need.

Ask Mode (Read-only)

The AI reads your files and answers questions. It cannot make any changes.

Use when: "What columns are in the competitor pricing dataset?" / "Where is the market share logic documented?"

Plan Mode (Think first)

The AI explores the problem, proposes an approach, and waits for your approval before doing anything.

Use when: "How should I structure a competitive landscape dashboard?" — you want options and trade-offs before committing.

Agent Mode (Act)

The AI takes full action — reads files, writes code, runs commands, calls APIs.

Use when: "Pull last week's sales CSVs and build a variance summary by region." The agent will find the data files, run the analysis, and save the output.

Quick rule: Ask = learn. Plan = decide. Agent = do.

Which mode to pick

flowchart TD
    Start[What do you need?] --> Q1{Just understand\nor explore?}
    Q1 -->|Yes| Ask["Ask Mode\n(read-only)"]
    Q1 -->|No| Q2{Want options and\ntrade-offs first?}
    Q2 -->|Yes| Plan["Plan Mode\n(think first)"]
    Q2 -->|No| Agent["Agent Mode\n(act)"]

    style Plan fill:#cce5ff,stroke:#333

G. What is Context?

Context is everything the AI knows about the current situation when it generates a response.

LLMs have no persistent memory. Each conversation starts fresh. Whatever you want the AI to know, you must include it in the context window — the "working memory" of a single session.

What counts as context?

  • The conversation history so far
  • Files you've shared or opened
  • Rules and instructions loaded at startup
  • Results of previous tool calls (e.g., search results, file contents)

Why it matters

ScenarioWithout contextWith context
"Summarize the competitor"Generic, outdated summaryBrief grounded in the filings and notes you shared
"Write the stakeholder update"Generic status emailUpdate tailored to your audience, project, and open risks

Context window = the maximum amount of text the AI can hold in memory at once. Measured in tokens (~1 token ≈ 0.75 words). GPT-4 handles ~128,000 tokens; Claude handles ~200,000.

When context is full, older information gets dropped — so keep prompts focused.

The context window (what fills up)

flowchart TB
    subgraph window ["Context window — finite working memory"]
        direction TB
        H[Conversation history]
        F[Files you @-mentioned]
        RI[Rules & instructions]
        TC[Tool call results]
    end

    window --> LLM[LLM uses all of this\nto generate the next reply]

    Full["⚠ When full → oldest entries\ndrop off silently"] -.-> window

    style window fill:#f8f9fa,stroke:#666

Everything in the box competes for the same space. A 50-page doc plus a long chat can push out your original instructions.

H. How Agents Use Skills, Rules, and Instructions

Here is how all the pieces fit together when an agent runs a task:

flowchart TD
    Task[You give a task] --> LoadI[Load Instructions]
    Task --> LoadR[Load Rules]
    Task --> MatchS{Matching skill?}
    MatchS -->|Yes| LoadS[Load Skill playbook]
    MatchS -->|No| Plan
    LoadI --> Plan[Make a plan]
    LoadR --> Plan
    LoadS --> Plan
    Plan --> Exec[Execute step by step\nvia tools]
    Exec --> Check{Irreversible\naction?}
    Check -->|Yes| Rules[Re-check rules]
    Rules --> Exec
    Check -->|No| Exec
    Exec --> Done[Return result to you]

    style Task fill:#d4edda,stroke:#333
    style Done fill:#d4edda,stroke:#333

Concrete example

Task: "Create Jira tickets from today's stakeholder workshop notes." (Business Analyst)

  1. Instructions tell the agent to be concise and ask if anything is unclear.
  2. Rules say never create duplicate tickets — so the agent searches Jira first.
  3. Skill "capture-tasks-from-workshop-notes" tells it: read the notes → find action items → look up assignee account IDs → create tickets with context.
  4. Agent executes all steps, then confirms what it created.

The result: You didn't have to tell the agent how to do any of this. The skill already knew. You just described what you wanted.

flowchart LR
    Notes[Workshop notes] --> Skill[Skill:\ncapture-tasks]
    Skill --> Search[Search Jira\nfor duplicates]
    Search --> Create[Create tickets\nwith assignees]
    Create --> Confirm[Confirm to you]

    style Skill fill:#cce5ff,stroke:#333

I. Prompt vs Agent — Which One to Use?

This is the most practical decision you'll make when using AI tools.

Use a prompt when…

The task is one shot — you describe it, the AI responds, you're done.

SituationExample prompt
Get an explanation"Explain how cohort retention analysis works in simple terms."
Draft something"Write a Microsoft Teams message summarizing tomorrow's competitive intel sync."
Transform content"Summarize this 10-K filing excerpt in 5 bullet points for executives."
Answer a question"What is the difference between a business analyst and a data analyst?"
Generate options"Give me 5 metrics to track on a competitor pricing dashboard."

Signal: You could hand the output directly to a person with no extra steps.

Build an agent when…

The task requires multiple steps, decisions, or tools — and doing it manually every time is tedious.

SituationWhat the agent doesPersona
Weekly competitive reportVisits 5 competitor websites → extracts updates → formats a report → saves to DriveCompetitive Intelligence Analyst
Triage research requestsReads new intake forms → classifies by type → checks Jira for duplicates → creates or updates issuesBusiness Analyst
Post-workshop actionsReads transcript → extracts action items → looks up assignees → creates Jira tasksBusiness Analyst
Weekly metrics digestPulls warehouse exports → validates row counts → builds variance tables → saves summary to DriveData Analyst
Competitive filing reviewReads new SEC filings → flags pricing or partnership changes → drafts an intel briefCompetitive Intelligence Analyst

Signal: You find yourself doing the same sequence of steps repeatedly, or the task involves reading/writing from multiple tools.

The decision flowchart

flowchart TD
    Start[New task] --> Q1{One step only?}
    Q1 -->|Yes| Prompt[Use a prompt]
    Q1 -->|No| Q2{Uses more than one\ntool or data source?}
    Q2 -->|No| Prompt
    Q2 -->|Yes| Q3{Will you repeat\nthis regularly?}
    Q3 -->|No| Prompt
    Q3 -->|Yes| Agent[Build an agent]

    style Prompt fill:#fff3cd,stroke:#333
    style Agent fill:#d4edda,stroke:#333

PromptAgent

Practical rule of thumb

If you can describe the entire task in one sentence and the AI can respond in one shot → prompt. If you'd need to write a checklist of steps for a new hire → agent.
